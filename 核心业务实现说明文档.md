## 2️⃣ 核心接口说明

### 2.1 内容管理接口

#### 2.1.1 创建内容
```
POST /api/content/create
{
  "title": "文章标题",
  "content": "文章内容",
  "creatorId": 2
}
返回: { "code": 200, "data": { "id": 1, "status": "DRAFT" } }
```
**说明**: 创建内容，初始状态为草稿。仅记录操作日志 CREATE_CONTENT。

#### 2.1.2 编辑内容
```
PUT /api/content/{contentId}?userId=2
{
  "title": "新标题",
  "content": "新内容"
}
返回: { "code": 200, "data": { "id": 1, "status": "DRAFT" } }
```
**校验规则**:
- ✅ 只能编辑自己创建的内容
- ✅ 只能编辑草稿(DRAFT)状态的内容
- ✅ 拒绝(REJECTED)状态的内容可以重新编辑

#### 2.1.3 删除内容
```
DELETE /api/content/{contentId}?userId=2
返回: { "code": 200, "data": "内容删除成功" }
```
**校验规则**:
- ✅ 只能删除自己创建的内容
- ✅ 只能删除草稿(DRAFT)状态的内容

#### 2.1.4 查询内容列表
```
GET /api/content/list?status=PENDING&creatorId=2&page=1&pageSize=10
返回: {
  "code": 200,
  "data": {
    "total": 5,
    "page": 1,
    "pageSize": 10,
    "data": [
      {
        "contentId": 1,
        "title": "文章标题",
        "status": "PENDING",
        "creatorId": 2,
        "creatorName": "operator",
        "createdAt": "2026-04-16T02:30:00",
        "updatedAt": "2026-04-16T02:31:00"
      }
    ]
  }
}
```
**查询参数**:
- `status` (可选): 内容状态筛选
- `creatorId` (可选): 创建人筛选
- `page` (默认1): 页码
- `pageSize` (默认10): 每页数量
- `startDate` (可选): 开始日期
- `endDate` (可选): 结束日期

### 2.2 审核管理接口

#### 2.2.1 审核处理
```
POST /api/content/audit
{
  "contentId": 1,
  "reviewerId": 3,
  "action": "APPROVED",  // APPROVED 或 REJECTED
  "comment": "审核意见"
}
返回: { "code": 200, "data": { "auditId": 1, "status": "APPROVED" } }
```
**校验规则**:
- ✅ 内容必须存在
- ✅ 内容状态必须为 PENDING(待审核)
- ✅ REJECTED 时 comment 必填
- ✅ 完整的错误提示

**状态转换**:
- APPROVED: PENDING → APPROVED
- REJECTED: PENDING → REJECTED

---

### 2.3 发布管理接口

#### 2.3.1 发布内容
```
POST /api/content/{contentId}/publish?userId=2
返回: { "code": 200, "data": { "id": 1, "status": "ONLINE" } }
```
**校验规则**:
- ✅ 只能发布审核通过(APPROVED)的内容
- ✅ 只能发布自己创建的内容

**状态转换**: APPROVED → ONLINE

#### 2.3.2 下线内容
```
POST /api/content/{contentId}/offline?userId=2
返回: { "code": 200, "data": { "id": 1, "status": "OFFLINE" } }
```
**校验规则**:
- ✅ 只能下线已上线(ONLINE)的内容
- ✅ 只能下线自己创建的内容

**状态转换**: ONLINE → OFFLINE

## 3️⃣ 内容状态流转说明

### 3.1 完整状态流转图

```
┌─────────────────────────────────────────────────────┐
│                  DRAFT (草稿)                        │
│                                                       │
│          ↓ 提交审核           ↑ 编辑修改             │
│      (SUBMIT_REVIEW)      (拒绝后编辑)              │
│          ↓                   ↑                       │
│    ┌──────────────┐          │                      │
│    │ PENDING (待审核)        │                      │
│    └──────────────┘          │                      │
│      ↙ (审核通过)   ↖ (审核拒绝)                    │
│     ↙               ↖                               │
│  APPROVED      REJECTED                             │
│  (审核通过)    (审核拒绝)                            │
│     ↓                                               │
│  发布(PUBLISH)                                      │
│     ↓                                               │
│  ONLINE (已上线)                                    │
│     ↓                                               │
│  下线(OFFLINE)                                      │
│     ↓                                               │
│  OFFLINE (已下线)                                   │
└─────────────────────────────────────────────────────┘
```

### 3.2 状态定义

| 状态 | 说明 | 可执行操作 |
|------|------|----------|
| **DRAFT** | 草稿 | 编辑、删除、提交审核 |
| **PENDING** | 待审核 | 审核通过、审核拒绝 |
| **APPROVED** | 审核通过 | 发布上线 |
| **REJECTED** | 审核拒绝 | 编辑修改、重新提交 |
| **ONLINE** | 已上线 | 下线 |
| **OFFLINE** | 已下线 | 无 |

### 3.3 非法状态转换拒绝示例

- ❌ DRAFT 直接转 ONLINE
- ❌ PENDING 状态重复提交
- ❌ OFFLINE 状态尝试发布
- ❌ 普通用户执行审核操作

## 4️⃣ 审核流程说明

### 4.1 审核流程图

```
运营人员创建内容(DRAFT)
        ↓
提交审核(PENDING)
        ↓
    审核员审核
    ↙      ↖
审核通过    审核拒绝
(APPROVED)  (REJECTED)
    ↓          ↓
发布上线    返回修改
(ONLINE)  (允许编辑+重新提交)
    ↓
可下线
(OFFLINE)
```

### 4.2 审核规则

**审核通过条件**:
- ✅ 内容状态必须为 PENDING
- ✅ 审核人必须存在
- ✅ 无需填写 comment (可选)

**审核拒绝条件**:
- ✅ 内容状态必须为 PENDING
- ✅ 审核人必须存在
- ✅ comment 必填(拒绝原因)

**拒绝后回流**:
- 审核拒绝后，内容状态变为 REJECTED
- 运营人员可编辑内容
- 编辑后可重新提交审核

## 5️⃣ 关键实现逻辑说明

### 5.1 操作日志记录

系统自动记录以下操作:

| 操作类型 | action | 触发条件 |
|---------|--------|--------|
| 创建内容 | CREATE_CONTENT | POST /api/content/create |
| 编辑内容 | UPDATE_CONTENT | PUT /api/content/{id} |
| 删除内容 | DELETE_CONTENT | DELETE /api/content/{id} |
| 提交审核 | SUBMIT_REVIEW | POST /api/content/{id}/submit-review |
| 审核通过 | APPROVE_CONTENT | POST /api/content/audit (action=APPROVED) |
| 审核拒绝 | REJECT_CONTENT | POST /api/content/audit (action=REJECTED) |
| 发布上线 | PUBLISH_CONTENT | POST /api/content/{id}/publish |
| 下线 | OFFLINE_CONTENT | POST /api/content/{id}/offline |

### 5.2 审核记录

每次审核都会产生一条 AuditRecord:

```java
{
  "id": 1,
  "contentId": 1,
  "reviewerId": 3,
  "status": "APPROVED",  // APPROVED 或 REJECTED
  "comment": "审核意见",
  "createdAt": "2026-04-16T02:30:00"
}
```

### 5.3 权限校验

**操作权限规则**:

| 操作 | 需要权限 | 校验逻辑 |
|------|--------|--------|
| 创建内容 | OPERATOR | 任何已认证用户 |
| 编辑内容 | OPERATOR | 创建人 + DRAFT状态 |
| 删除内容 | OPERATOR | 创建人 + DRAFT状态 |
| 提交审核 | OPERATOR | 创建人 + DRAFT/REJECTED |
| 审核处理 | REVIEWER | 审核人存在 |
| 发布上线 | OPERATOR | 创建人 + APPROVED |
| 下线 | OPERATOR | 创建人 + ONLINE |

### 5.4 错误处理

系统统一使用 BusinessException 处理业务错误:

```java
throw new BusinessException(404, "内容不存在");
throw new BusinessException(403, "只能编辑自己创建的内容");
throw new BusinessException(400, "只有草稿状态的内容才能编辑");
```

**错误码说明**:
- 400: 参数错误或业务规则违反
- 403: 权限不足
- 404: 资源不存在
- 500: 系统内部错误

## 6️⃣ 数据库表关系

### 6.1 表间关系图

```
user (用户表)
  ├─→ role (角色表)
  └─→ content (通过 creator_id)
  └─→ audit_record (通过 reviewer_id)
  └─→ operation_log (通过 user_id)

content (内容表)
  ├─→ user (creator_id)
  └─→ audit_record (content_id)
  └─→ operation_log (target_id)

audit_record (审核记录表)
  ├─→ content (content_id)
  └─→ user (reviewer_id)

operation_log (操作日志表)
  └─→ user (user_id)
```

### 6.2 关键字段

- `content.status`: 内容整体状态
- `audit_record.status`: 单次审核结果
- `operation_log.action`: 操作类型
- `created_at`, `updated_at`: 时间追踪
