【第1章】系统模块划分
================================================================================

系统分为三个核心模块：

【1. 管理模块（admin）】
  职责: 用户、角色、权限管理
  包含: User、Role、Permission、PermissionManageService、
        PermissionCheckService、PermissionController
  API: /api/admin/*
  权限: ADMIN

【2. 内容模块（content）】
  职责: 内容创建、编辑、删除、查询
  包含: Content、OperationLog、ContentService、ContentController
  API: /api/content/*
  权限: OPERATOR
  状态: DRAFT → PENDING → APPROVED → ONLINE → OFFLINE

【3. 审核模块（audit）】
  职责: 内容审核流程、审核结果记录
  包含: AuditRecord、AuditService、AuditController
  API: /api/audit/*
  权限: REVIEWER
  操作: 审核通过/拒绝，记录审核意见

【第2章】核心接口说明
================================================================================

### 内容管理接口（7个）

1. POST /api/content/create - 创建内容
   权限: OPERATOR
   状态: 无 → DRAFT
   
2. PUT /api/content/{id}/update - 编辑内容
   权限: OPERATOR
   状态: DRAFT only
   
3. DELETE /api/content/{id} - 删除内容
   权限: OPERATOR
   状态: DRAFT only
   
4. POST /api/content/submit-review - 提交审核
   权限: OPERATOR
   状态: DRAFT/REJECTED → PENDING
   
5. POST /api/content/publish - 发布上线
   权限: OPERATOR
   状态: APPROVED → ONLINE
   
6. POST /api/content/offline - 下线内容
   权限: OPERATOR
   状态: ONLINE → OFFLINE
   
7. GET /api/content/list - 查询内容列表
   权限: 无限制
   支持: 分页、按状态筛选、按创建者筛选、按时间范围筛选

### 审核接口（1个）

1. POST /api/audit/audit - 审核内容
   权限: REVIEWER
   操作: 通过或拒绝
   状态: PENDING → APPROVED/REJECTED
   记录: 审核意见、审核时间

### 管理接口（7个）

1. POST /api/admin/users - 创建用户
   权限: ADMIN
   
2. PUT /api/admin/users/{id} - 更新用户
   权限: ADMIN
   
3. DELETE /api/admin/users/{id} - 删除用户
   权限: ADMIN
   
4. GET /api/admin/users - 查询用户
   权限: 无限制
   
5. POST /api/admin/permissions - 创建权限
   权限: ADMIN
   
6. GET /api/admin/permissions - 查询权限
   权限: 无限制
   
7. GET /api/admin/stats - 系统统计
   权限: 无限制

总计: 15个接口

【第3章】内容状态流转说明
================================================================================

### 状态定义

DRAFT（草稿）
  - 创建后的初始状态
  - 可编辑、可删除、可提交审核

PENDING（待审核）
  - 运营提交审核后的状态
  - 不可编辑，等待审核

APPROVED（审核通过）
  - 审核员审核通过后的状态
  - 可发布上线

REJECTED（审核拒绝）
  - 审核员拒绝后的状态
  - 可编辑，编辑后状态变为DRAFT可重新提交

ONLINE（已上线）
  - 发布后的状态
  - 已对用户可见
  - 可下线

OFFLINE（已下线）
  - 下线后的状态
  - 最终状态，不可再操作

### 状态流转规则

创建 → DRAFT
编辑 → 保持DRAFT
删除 → 删除内容
提交审核 → 从DRAFT/REJECTED变为PENDING
审核通过 → 从PENDING变为APPROVED
审核拒绝 → 从PENDING变为REJECTED
编辑被拒内容 → REJECTED变为DRAFT
重新提交 → 从DRAFT变为PENDING
发布 → 从APPROVED变为ONLINE
下线 → 从ONLINE变为OFFLINE

### 非法状态流转防止

系统在每个状态转换前都进行校验：

- 编辑: 仅DRAFT状态可编辑
- 删除: 仅DRAFT状态可删除
- 提交: 仅DRAFT/REJECTED状态可提交
- 发布: 仅APPROVED状态可发布
- 下线: 仅ONLINE状态可下线
- 重复提交: PENDING状态不允许再提交（防止重复）

状态不符合条件时，系统返回400错误。

【第4章】审核流程说明
================================================================================

### 审核流程

```
运营人员提交内容(PENDING)
    ↓
审核员收到审核任务
    ↓
审核员选择: 通过 或 拒绝
    ↓           ↓
  通过      拒绝(需填写原因)
    ↓           ↓
 APPROVED   REJECTED
    ↓           ↓
 可发布     运营重新编辑
    ↓       
 发布上线
    ↓
 ONLINE
```

### 审核操作

1. 审核通过（APPROVED）
   - 内容状态: PENDING → APPROVED
   - 审核记录: 记录审核员、审核时间、审核结果
   - 后续操作: 运营可发布上线

2. 审核拒绝（REJECTED）
   - 内容状态: PENDING → REJECTED
   - 审核记录: 记录审核员、审核时间、拒绝原因
   - 后续操作: 运营修改后重新提交

### 审核权限控制

- 仅REVIEWER角色可进行审核
- 审核人必须存在
- 内容必须存在
- 内容必须处于PENDING状态
- 拒绝时必须填写拒绝原因

### 审核留痕

每一次审核操作都被记录：
- 审核记录表（audit_record）记录审核历史
- 操作日志表（operation_log）记录所有操作
- 便于追溯和审计

【第5章】关键实现逻辑
================================================================================

### 1. 权限校验

权限检查通过以下方式实现：

Method: checkPermission(userId, requiredRole)
- 查询用户信息
- 获取用户角色
- 比较是否符合要求
- 不符合返回403异常

应用场景：
- 创建用户: 需要ADMIN
- 创建内容: 需要OPERATOR
- 审核内容: 需要REVIEWER

### 2. 状态流转校验

在每个状态转换前进行校验：

```java
if (content.getStatus() != ContentStatus.DRAFT) {
    throw new BusinessException(400, "只有草稿状态的内容才能编辑");
}
```

所有状态转换都有对应的校验，防止非法操作。

### 3. 操作权限校验

确保用户只能操作自己的内容：

```java
if (!permissionService.isContentCreator(userId, content.getCreator().getId())) {
    throw new BusinessException(403, "只能编辑自己创建的内容");
}
```

### 4. 时间格式化

所有时间字段显示格式为 yyyy-MM-dd HH:mm:ss（精确到秒）

实现：DateTimeUtil.format(LocalDateTime)

### 5. 操作日志记录

所有关键操作都被记录：

```java
operationLogService.log(userId, "CREATE_CONTENT", contentId);
```

操作包括：
- CREATE_CONTENT
- UPDATE_CONTENT
- DELETE_CONTENT
- SUBMIT_REVIEW
- APPROVE_CONTENT
- REJECT_CONTENT
- PUBLISH_CONTENT
- OFFLINE_CONTENT
- CREATE_USER
- UPDATE_USER
- DELETE_USER
- CREATE_PERMISSION
- DELETE_PERMISSION

### 6. 分页查询

GET /api/content/list 实现分页查询：
- 支持按状态筛选
- 支持按创建者筛选
- 支持按时间范围筛选
- 返回总数、当前页码、每页数量、数据列表

### 7. 异常处理

系统定义了统一的异常处理：

- BusinessException(statusCode, message)
  - 400: 参数错误、状态错误
  - 403: 权限不足
  - 404: 资源不存在

- GlobalExceptionHandler 捕获所有异常
- 返回统一的错误响应格式
