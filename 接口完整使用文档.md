【第1章】内容管理模块 (/api/content)
================================================================================

### 1.1 创建内容
Method: POST
URL: http://localhost:8080/api/content/create
Permission: OPERATOR

Body (JSON):
{
  "title": "Spring Boot 最佳实践",
  "content": "这是内容详情...",
  "creatorId": 2
}

Response:
{
  "contentId": 1,
  "status": "DRAFT"
}

---

### 1.2 编辑内容
Method: PUT
URL: http://localhost:8080/api/content/{contentId}/update?userId=2
Permission: OPERATOR (仅创建者)
Status: DRAFT only

Path Params:
  contentId = 1

Query Params:
  userId = 2

Body (JSON):
{
  "title": "新标题",
  "content": "新内容..."
}

---

### 1.3 删除内容
Method: DELETE
URL: http://localhost:8080/api/content/{contentId}?userId=2
Permission: OPERATOR (仅创建者)
Status: DRAFT only

Path Params:
  contentId = 1

Query Params:
  userId = 2

---

### 1.4 提交审核
Method: POST
URL: http://localhost:8080/api/content/submit-review?contentId=1&userId=2
Permission: OPERATOR
State: DRAFT/REJECTED → PENDING

Query Params:
  contentId = 1
  userId = 2

---

### 1.5 发布内容
Method: POST
URL: http://localhost:8080/api/content/publish?contentId=1&userId=2
Permission: OPERATOR
State: APPROVED → ONLINE

Query Params:
  contentId = 1
  userId = 2

---

### 1.6 下线内容
Method: POST
URL: http://localhost:8080/api/content/offline?contentId=1&userId=2
Permission: OPERATOR
State: ONLINE → OFFLINE

Query Params:
  contentId = 1
  userId = 2

---

### 1.7 查询内容列表
Method: GET
URL: http://localhost:8080/api/content/list?page=1&pageSize=10&status=PENDING
Permission: 无限制

Query Params:
  page = 1 (default: 1)
  pageSize = 10 (default: 10)
  status = PENDING (optional: DRAFT/PENDING/APPROVED/REJECTED/ONLINE/OFFLINE)
  creatorId = 2 (optional)
  startDate = 2026-01-01 (optional, format: yyyy-MM-dd)
  endDate = 2026-12-31 (optional, format: yyyy-MM-dd)

Response:
{
  "total": 5,
  "page": 1,
  "pageSize": 10,
  "data": [...]
}

【第2章】审核模块 (/api/audit)
================================================================================

### 2.1 审核内容
Method: POST
URL: http://localhost:8080/api/audit/audit
Permission: REVIEWER
State: PENDING → APPROVED or REJECTED

Headers:
  Content-Type = application/json

Body (JSON) - 审核通过:
{
  "contentId": 1,
  "reviewerId": 4,
  "action": "APPROVED",
  "comment": "内容质量不错，通过审核"
}

Body (JSON) - 审核拒绝:
{
  "contentId": 1,
  "reviewerId": 4,
  "action": "REJECTED",
  "comment": "需要补充更多技术细节"
}

Response:
{
  "id": 1,
  "contentId": 1,
  "status": "APPROVED",
  "comment": "..."
}

【第3章】管理模块 (/api/admin)
================================================================================

### 3.1 创建用户
Method: POST
URL: http://localhost:8080/api/admin/users
Permission: ADMIN

Headers:

Body (JSON):

  "username": "operator1",
  "password": "password123",
  "roleId": 2
}

---

### 3.2 更新用户
Method: PUT
URL: http://localhost:8080/api/admin/users/{userId}
Permission: ADMIN

Headers:

  "password": "newpassword123",
  "roleId": 3

---

### 3.3 删除用户
Method: DELETE
URL: http://localhost:8080/api/admin/users/{userId}
Permission: ADMIN

Headers:

Path Params:
  userId = 2

【第4章】HTTP状态码
================================================================================

200: 成功
400: 参数错误或业务规则不符
403: 无权限
404: 资源不存在
500: 服务器错误

================================================================================
【第5章】内容生命周期
================================================================================

DRAFT → 编辑 → DRAFT / 提交审核 → PENDING
         ↓                        ↓
        删除                  审核通过/拒绝
                              ↓        ↓
                           APPROVED REJECTED
                              ↓        ↓
                            发布     编辑 → 重新提交
                              ↓
                            ONLINE
                              ↓
                            下线
                              ↓
                            OFFLINE

================================================================================
【第6章】用户角色
================================================================================

ADMIN (id=1): 用户/角色/权限管理，需要 X-User-Id=1
OPERATOR (id=2,3): 内容创建/编辑/审核提交
REVIEWER (id=4,5): 内容审核
