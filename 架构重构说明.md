## 🎯 模块职责

### 1️⃣ 权限模块 (Permission Module)

**职责**：
- 管理用户、角色、权限
- 提供权限校验接口
- 控制用户访问范围

**核心类**：
- `PermissionService.java` - 权限检查服务
  - `canCreateContent()` - 检查是否能创建内容
  - `canAuditContent()` - 检查是否能审核
  - `isContentCreator()` - 检查是否是创建人

**API 端点**：
- 用户管理、角色管理、权限分配 (可扩展)

---

### 2️⃣ 内容模块 (Content Module)

**职责**：
- 提供内容创建、修改、查询功能
- 实现内容生命周期管理
- 记录操作日志

**核心类**：
- `ContentService.java` - 内容业务逻辑
  - `createContent()` - 创建内容
  - `updateContent()` - 编辑内容
  - `deleteContent()` - 删除内容
  - `submitForReview()` - 提交审核
  - `publishContent()` - 发布上线
  - `offlineContent()` - 下线
  - `listContent()` - 查询列表

- `OperationLogService.java` - 操作日志记录
  - `log()` - 记录每次操作

**API 端点**：
- `POST /api/content/create` - 创建内容
- `PUT /api/content/{contentId}/update` - 编辑内容
- `DELETE /api/content/{contentId}` - 删除内容
- `POST /api/content/submit-review` - 提交审核
- `POST /api/content/publish` - 发布
- `POST /api/content/offline` - 下线
- `GET /api/content/list` - 查询列表

---

### 3️⃣ 审核模块 (Audit Module)

**职责**：
- 管理内容审核流程
- 记录审核结果
- 实现通过/驳回及回流逻辑

**核心类**：
- `AuditService.java` - 审核业务逻辑
  - `auditContent()` - 执行审核
    - 通过：PENDING → APPROVED
    - 拒绝：PENDING → REJECTED

**特殊逻辑**：
- 拒绝后，内容回到 REJECTED 状态
- 用户可以重新提交审核（通过 ContentService.submitForReview）
- 这实现了 REJECTED → PENDING 的回流逻辑

**API 端点**：
- `POST /api/audit/audit` - 执行审核

## 🔄 业务流程

### 内容生命周期

```
1. 创建 (CREATE_CONTENT)
   ├─ 权限检查：用户是否是 OPERATOR 或 ADMIN
   ├─ 状态：DRAFT
   └─ 操作日志：CREATE_CONTENT

2. 编辑 (UPDATE_CONTENT) [可选]
   ├─ 权限检查：只能编辑自己的
   ├─ 状态检查：只能编辑 DRAFT
   └─ 操作日志：UPDATE_CONTENT

3. 删除 (DELETE_CONTENT) [可选]
   ├─ 权限检查：只能删除自己的
   ├─ 状态检查：只能删除 DRAFT
   └─ 操作日志：DELETE_CONTENT

4. 提交审核 (SUBMIT_REVIEW)
   ├─ 权限检查：只能提交自己的
   ├─ 状态转换：DRAFT/REJECTED → PENDING
   └─ 操作日志：SUBMIT_REVIEW

5. 审核 (APPROVE_CONTENT / REJECT_CONTENT)
   ├─ 权限检查：用户是否是 REVIEWER 或 ADMIN
   ├─ 状态检查：必须是 PENDING
   ├─ 通过：PENDING → APPROVED
   ├─ 拒绝：PENDING → REJECTED（可回流）
   └─ 操作日志：APPROVE_CONTENT / REJECT_CONTENT

6. 发布 (PUBLISH_CONTENT)
   ├─ 权限检查：只能发布自己的
   ├─ 状态检查：只能发布 APPROVED
   ├─ 状态转换：APPROVED → ONLINE
   └─ 操作日志：PUBLISH_CONTENT

7. 下线 (OFFLINE_CONTENT)
   ├─ 权限检查：只能下线自己的
   ├─ 状态检查：只能下线 ONLINE
   ├─ 状态转换：ONLINE → OFFLINE（最终状态）
   └─ 操作日志：OFFLINE_CONTENT
```
